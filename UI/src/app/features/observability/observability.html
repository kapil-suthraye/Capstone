Metric Explanations

Definitions and usage guidance for the metrics displayed above.

Request Latency

Avg is the arithmetic mean. P95 indicates that 95 % of requests complete faster than this threshold — a better indicator of user-perceived performance. A P95 under 500 ms is considered excellent for interactive workflows.

Error Rate

The ratio of 5xx server errors to total requests. A healthy system runs at < 1 %. Transient spikes are normal, but sustained elevated error rates indicate backend instability and warrant investigation.

RAGAS Scores

Faithfulness checks factual grounding. Relevancy checks answer alignment with the question. Precision and Recall measure retrieval quality. Utilisation checks whether the model actually used the retrieved context. All metrics range from 0 (worst) to 1 (best).

Verdict Distribution

Valid: criterion is met with high confidence. Doubtful: evidence is weak or conflicting; requires manual review. Insufficient Evidence: no relevant documentation found. This distribution helps spot documentation gaps and recurring review bottlenecks.
